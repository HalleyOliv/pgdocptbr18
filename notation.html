3. Convenções #

As seguintes convenções são usadas nas sinopses dos comandos: os colchetes ([ e ]) indicam partes opcionais. As chaves ({ e }) e as barras verticais (|) indicam que deve ser escolhida uma das alternativas. Os pontos (...) significam que o elemento precedente pode ser repetido. Todos os outros símbolos, incluindo os parênteses, devem ser interpretados literalmente.

Quando fica mais claro, os comandos SQL são precedidos pelo prompt =>, e os comandos para o interpretador de comandos (shell) são precedidos pelo prompt $. Entretanto, normalmente os prompts não são mostrados.

Geralmente o Administrador é a pessoa responsável pela instalação e funcionamento do servidor. O Usuário pode ser qualquer pessoa usando, ou querendo usar, qualquer parte do sistema PostgreSQL. Esses termos não devem ser interpretados de forma muito estrita; esta documentação não estabelece presunções com relação aos procedimentos de administração do sistema.
