1.1. Instalação #

Antes de usar o PostgreSQL é necessário instalá-lo, obviamente. É possível que o PostgreSQL já esteja instalado no seu ambiente, seja porque está incluído na distribuição do sistema operacional, ou porque o administrador do sistema fez a instalação. Se for esse o caso, devem ser obtidas informações na documentação do sistema operacional, ou com o administrador do sistema, sobre como acessar o PostgreSQL.

Se não houver certeza de que o PostgreSQL está disponível, ou se pode ser utilizado para as suas necessidades, então você mesmo poderá fazer a instalação. Fazer isso não é difícil, podendo ser um bom exercício. O PostgreSQL pode ser instalado por qualquer usuário sem privilégios, porque não é necessário nenhum acesso de superusuário (root).

Se você mesmo for instalar o PostgreSQL, então leia antes o Capítulo 17 para conhecer as instruções de instalação e, depois, retorne para esse capítulo quando a instalação estiver concluída. Certifique-se de seguir cuidadosamente a seção sobre a configuração das variáveis de ambiente apropriadas.

Se o administrador do sistema não fez a configuração da maneira padrão, talvez seja necessário algum trabalho adicional. Por exemplo, se a máquina servidora de banco de dados for uma máquina remota, será necessário definir a variável de ambiente PGHOST com o nome da máquina servidora de banco de dados. Também, talvez tenha que ser definida a variável de ambiente PGPORT. A regra básica é essa: quando se tenta iniciar um programa e ele informa que não está conseguindo se conectar com o servidor de banco de dados, deve ser consultado o administrador do servidor ou, caso seja você mesmo, a documentação, para ter certeza de que o ambiente está configurado de maneira apropriada. Caso não tenha entendido o parágrafo anterior, então, por favor, leia a próxima seção.
